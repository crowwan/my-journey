**작성자**: kimjin-wan + Claude

---

## 1. 핵심 요약

여행 계획을 **월간 캘린더**로 시각화하는 페이지. Mac 캘린더 스타일로 여행 기간이 가로 바(bar)로 표시되며, 클릭하면 해당 여행 상세로 이동. 기존 캘린더 추가(외부 연동) 기능은 제거.

---

## 2. 현재 상태 (AS-IS)

- 홈에서 여행 목록을 카드 형태로만 확인 가능
- 여행 기간이 겹치거나 연속인지 시각적으로 파악 어려움
- 캘린더 추가 기능 존재 여부 확인 필요

### Mac 캘린더 스타일 특징
- 여행 기간이 **가로 바(bar)**로 표시 (여러 날에 걸침)
- 여행별 **고유 색상** (Trip의 days[0].color 활용)

- 월 단위 탐색 (< > 화살표)
- 오늘 날짜 하이라이트

## 4. 구현 설계

### 4.1 라우트

- `/calendar` — 캘린더 뷰 페이지
- `src/app/calendar/page.tsx`

### 4.2 컴포넌트 구조

```
CalendarPage
├── CalendarHeader (월 표시 + < > 네비게이션)
├── CalendarGrid
│   ├── WeekdayHeader (일~토)
│   └── CalendarRow × 4~6주
│       └── CalendarCell × 7일
│           └── TripBar (여행 바, 여러 셀에 걸침)
└── (선택) 하단 여행 목록 요약
```

### 4.3 여행 바 렌더링 로직

```
1. useTripStore에서 전체 trips 가져오기
2. 현재 표시 월의 날짜 범위 계산
3. 각 Trip의 startDate ~ endDate가 현재 월과 겹치는지 확인
4. 겹치는 Trip을 주(week) 단위로 분할
   - 한 주를 넘어가면 다음 주에 이어서 바 표시
5. 같은 주에 여러 Trip이 있으면 세로로 쌓기 (Mac 캘린더처럼)
6. 바 시작/끝 위치를 grid-column으로 계산
```

### 4.4 네비게이션

- 헤더: "2026년 4월" + < 이전 월 / 다음 월 > 버튼
- 오늘로 돌아가기 버튼
- 홈에서 캘린더 진입점 추가 (탭 또는 아이콘 버튼)

### 4.5 스타일

- 디자인 시스템 준수 (`docs/design-system.md`)
- 여행 바 색상: Trip의 `days[0].color` 사용 (없으면 primary)
- 오늘 날짜: primary 색상 동그라미 또는 배경
- 주말(토/일): 약간 다른 배경색
- 반응형: 모바일에서도 월간 그리드 유지 (셀 크기 축소)

## 6. 수용 기준

- 월간 캘린더에 여행 기간이 Mac 캘린더 스타일 바로 표시된다
- 여행이 주를 넘어가면 다음 줄에 이어서 바가 표시된다
- 바 클릭 시 해당 여행 상세 페이지로 이동한다
- 이전/다음 월 탐색이 가능하다
- 오늘 날짜가 시각적으로 구분된다
- 모바일에서도 정상 표시된다
