# 여행 상세 인라인 편집 — 종합 계획서

**날짜**: 2026-03-15
**상태**: Phase 1 구현 완료 → 섹션별 편집으로 리팩토링
**작성자**: kimjin-wan + Claude

---

## 1. 핵심 요약

여행 상세 페이지에서 **AI 없이 사용자가 직접** 항목을 수정/추가/삭제할 수 있는 인라인 편집 기능. **섹션별 독립 편집** 방식으로, 각 섹션(체크리스트, 항공편, 일정 등)에 개별 편집 버튼이 있고, 해당 섹션만 편집 모드로 전환된다.

---

## 2. 설계 결정 변경 (v2)

### D1: 전체 편집 → 섹션별 편집 (변경)

```
변경 전 (전체 편집):
  HeroSection "편집" → 모든 섹션 편집 → 상단 EditBar "저장"/"취소"

변경 후 (섹션별 편집):
  각 섹션 헤더에 ✏️ 버튼 → 해당 섹션만 편집 → 섹션 내 "저장"/"취소"
  다른 섹션은 읽기 모드 유지
```

**이유**: 체크리스트만 수정하려는데 전체 편집 모드는 과도. 수정 범위가 명확해 실수 위험 감소.

### D2: useEditStore 단순화
- `editingSection: string | null` — 현재 편집 중인 섹션 식별자
- `editingTrip: Trip | null` — 편집 중인 Trip 복사본 (기존 유지)
- HeroSection "편집" 버튼 + 전체 EditBar 제거

### D3: AI 수정과의 공존
- 섹션 편집 중에는 "AI로 수정" 버튼 비활성화
- 반대로 AI Split View 중에는 섹션 편집 버튼 비활성화

## 3. 섹션별 편집 UI 패턴

```
읽기 모드:
┌─ 체크리스트 ──────────────── [✏️] ─┐
│ ☑ 여권                             │
│ ☐ 충전기                           │
│ ☐ 세면도구                         │
└────────────────────────────────────┘

편집 모드:
┌─ 체크리스트 ──── [저장] [취소] ─┐
│ [여권         ] [−]              │
│ [충전기       ] [−]              │
│ [세면도구     ] [−]              │
│ [+ 항목 추가]                    │
└────────────────────────────────────┘
```

각 섹션 SectionTitle 옆에 편집 아이콘 → 클릭 시 해당 섹션만 편집 UI 전환.

- TripViewer: editingTrip 분기 유지 (편집 중인 섹션이 있으면 editingTrip 사용)

## 6. 수용 기준

- 각 섹션 헤더에 편집 아이콘이 있다
- 편집 아이콘 클릭 시 해당 섹션만 편집 모드로 전환된다
- 다른 섹션은 읽기 모드를 유지한다
- 섹션 내 "저장" 클릭 → localStorage 반영
- 섹션 내 "취소" 클릭 → 변경 폐기
- 한 번에 하나의 섹션만 편집 가능하다
