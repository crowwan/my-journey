# 여행 상세 인라인 편집 — 종합 계획서

**날짜**: 2026-03-15
**상태**: Draft
**작성자**: kimjin-wan + Claude

---

## 1. 핵심 요약

여행 상세 페이지에서 **AI 없이 사용자가 직접** 항목을 수정/추가/삭제할 수 있는 인라인 편집 기능. AI 수정은 대규모 변경(일정 재구성, 목적지 추가)에 적합하고, 간단한 수정(항공편 시간, 체크리스트 추가)은 직접 편집이 훨씬 빠르고 자연스럽다.

---

## 2. 현재 상태 (AS-IS)

- 여행 데이터 수정은 **AI 대화로만** 가능
- 체크리스트 체크/해제만 직접 조작 가능 (localStorage 기반)
- 항공편 시간 1분 수정에도 AI 호출 필요 → 비효율
- 숙소 이름 오타 수정에도 AI 호출 필요

## 3. 목표 상태 (TO-BE)

### 편집 모드 전환

```
여행 상세 페이지 (읽기 모드)
    ↓ "편집" 버튼 클릭
여행 상세 페이지 (편집 모드)
    - 수정 가능한 필드에 편집 UI 표시
    - 항목 추가/삭제 버튼 표시
    - 상단에 "저장" / "취소" 버튼
    ↓ "저장" 클릭
useTripStore.saveTrip() → localStorage 저장
```

### AI 수정과의 역할 분담

| 수정 유형 | 방식 | 이유 |
|----------|------|------|
| 항공편 시간/편명 | **직접 편집** | 단순 값 변경 |
| 숙소 정보 | **직접 편집** | 단순 값 변경 |
| 체크리스트 추가/삭제/수정 | **직접 편집** | 빈번하고 개인화된 작업 |
| 사전 준비 추가/삭제/수정 | **직접 편집** | 동일 |
| 일정 아이템 시간/제목 수정 | **직접 편집** | 미세 조정 |
| 일정 아이템 추가/삭제 | **직접 편집** | 단순 CRUD |
| 맛집 이름/설명 수정 | **직접 편집** | 단순 값 변경 |
| Day 전체 재구성 | **AI 수정** | 동선 최적화 필요 |
| 여행 기간/목적지 변경 | **AI 수정** | 전체 재설계 필요 |
| 새 Day 일정 생성 | **AI 수정** | 장소 정보 필요 |

## 4. 구현 설계

### 4.1 편집 모드 상태 관리

```typescript
// useUIStore 또는 별도 useEditStore
interface EditState {
  isEditMode: boolean;
  editingTrip: Trip | null;  // 편집 중인 Trip 복사본 (원본 보존)
  enterEditMode: (trip: Trip) => void;
  exitEditMode: () => void;
  updateField: (path: string, value: unknown) => void;
  saveEdit: () => void;      // editingTrip → useTripStore.saveTrip
  cancelEdit: () => void;    // editingTrip 폐기, 원본 복원
}
```

### 4.2 섹션별 편집 UI

#### 항공편 (SummaryTab)
- 각 필드 탭 → 인라인 input으로 전환
- 편집 가능: departure, arrival, departureTime, arrivalTime, date, duration
- 항공편 추가/삭제 버튼

#### 숙소 (SummaryTab)
- name, address, area, nearbyStations 인라인 편집
- nearbyStations: 태그 형태 추가/삭제

#### 날씨 (SummaryTab)
- 편집 불필요 (AI 생성 데이터, 참고용)

#### 일정 아이템 (ScheduleTab)
- 각 타임라인 아이템: time, title, description 인라인 편집
- 드래그로 순서 변경 (선택사항)
- 아이템 추가 (+) / 삭제 (스와이프 또는 - 버튼)
- type 변경 (spot/food/move/default 드롭다운)

#### 맛집 (GuideTab)
- name, description, priceRange, rating 인라인 편집
- 맛집 추가/삭제

#### 교통 (GuideTab)
- homeToHotel: title, subtitle 편집
- 단계 추가/삭제
- passes, intercityRoutes: 편집 가능하나 우선순위 낮음

#### 예산 (GuideTab)
- items: label, amount, percentage 편집
- total: 자동 재계산 또는 수동 수정
- 항목 추가/삭제

#### 체크리스트 (ChecklistTab)
- 기존 체크 기능 유지
- **항목 추가**: 카테고리 하단 + 버튼
- **항목 삭제**: 스와이프 또는 - 버튼
- **항목 수정**: 이름 탭 → 인라인 편집
- **카테고리 추가/삭제**

#### 사전 준비 (ChecklistTab)
- title, description 인라인 편집
- 항목 추가/삭제
- 순서 변경 (order)

### 4.3 편집 UI 패턴

```
읽기 모드:

│ ✈️ 인천 → 나리타         │

└─────────────────────────┘
```

## 6. 수용 기준

- "편집" 버튼으로 편집 모드 진입/해제가 가능하다
- 편집 모드에서 수정 가능한 필드가 시각적으로 구분된다
- 체크리스트에 항목 추가/삭제/수정이 가능하다
- 항공편 시간/편명을 직접 수정할 수 있다
- "저장" 클릭 시 변경 사항이 localStorage에 반영된다
- "취소" 클릭 시 변경 사항이 폐기된다
- AI 수정("AI로 수정")과 직접 편집("편집")이 공존한다

## 7. 설계 결정

### D1: 편집 모드 vs 항상 편집 가능
- **편집 모드 토글 방식 선택** — 읽기/편집 분리로 실수 방지, UI 깔끔
- 체크리스트 체크/해제만 항상 가능 (기존 동작 유지)

### D2: 편집 중 데이터 관리
- **Trip 복사본으로 작업** — 원본은 저장 전까지 보존
- 취소 시 복사본 폐기, 저장 시 복사본 → 원본 덮어쓰기

### D3: AI 수정과의 공존
- HeroSection에 "편집" (직접) + "AI로 수정" (대화) 버튼 병행
- 편집 모드 중 AI 수정 불가 (충돌 방지)
