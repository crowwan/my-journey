## 1. 핵심 요약

현재 예산은 **카테고리별 총액**만 보여주는 단순 구조. 사용자가 실제 여행 예산을 계획하고 관리하기엔 부족하다. 숫자 기반 입력, 통화 자동 변환, Day별 예산 분해, 도넛 차트 시각화 등으로 고도화한다.

---

## 2. 현재 상태 분석 (AS-IS)

### 데이터 구조
```typescript
BudgetItem {
  icon: string;       // 이모지 ("🚆")
  label: string;      // "교통"
  detail: string;     // "전철, 버스 등"
  amount: string;     // "¥3,000" ← 통화+금액 합쳐진 문자열
  percentage: number;  // 30 ← 저장 시 자동 계산
  color: string;      // "#3b82f6"
}

BudgetRange {
  min/max: string;     // "¥50,000" / "¥80,000"
  minKRW/maxKRW: string; // "약 50만원" / "약 80만원"
}
```

### 문제점

| # | 문제 | 영향 |
|---|------|------|
| P1 | amount가 문자열 → 합계/비율 계산이 정규식 파싱 의존 | 부정확, 오류 가능 |
| P2 | 통화 정보가 금액 문자열에 혼합 → 환율 계산 불가 | 원화 환산 수동 입력 |
| P3 | 카테고리 총액만 → Day별 얼마 쓰는지 모름 | 일별 예산 배분 불가 |
| P4 | 비율 바만 → 전체 분포 한눈에 파악 어려움 | 도넛/파이 차트 없음 |
| P5 | total이 수동 입력 → 항목 합계와 불일치 가능 | 데이터 불일치 |
| P6 | 편집 시 비율 자동 계산이 저장 시점에만 → 실시간 피드백 없음 | UX 부족 |

## 3. 목표 상태 (TO-BE)

### 3.1 데이터 구조 개선

```typescript
// 변경된 BudgetItem
BudgetItem {
  icon: string;
  label: string;
  detail: string;
  amount: number;       // ← 숫자로 변경 (3000)
  currency: string;     // ← 통화 분리 ("JPY")
  percentage: number;   // 자동 계산
  color: string;
}

// 변경된 BudgetSection
BudgetSection {
  items: BudgetItem[];
  currency: string;      // ← 기본 통화 ("JPY")
  exchangeRate?: number; // ← 환율 (1 JPY = ? KRW)
  total: BudgetTotal;    // ← 자동 계산
  tips: string[];
}

// 변경된 Total (자동 계산)
BudgetTotal {
  amount: number;        // items 합계
  amountKRW: number;     // 환율 적용
  currency: string;      // 기본 통화
}
```

## 5. 설계 결정

### D1: amount 타입 변경 (string → number)
- **이유**: 계산/비교가 가능해짐, 통화 분리
- **마이그레이션**: 기존 `"¥3,000"` → 숫자 `3000` + currency `"JPY"` 파싱
- **위험**: 기존 localStorage 데이터 호환 → 로드 시 자동 마이그레이션

### D2: 도넛 차트 — SVG 직접 구현
- **이유**: 외부 라이브러리(chart.js, recharts) 없이 가볍게
- SVG `<circle>` + `stroke-dasharray`로 구현
- 각 항목 색상 적용

### D3: 환율 — 수동 입력
- **이유**: 실시간 API 호출은 프로토타입에서 과도
- 사용자가 환율 직접 입력 → 원화 자동 환산
- 향후: 환율 API 연동 가능

### D4: 하위 호환
- 기존 문자열 amount를 가진 Trip 로드 시 → 자동 파싱/변환
- 새 필드(currency, exchangeRate)가 없으면 기본값 적용

## 6. 수용 기준

- 예산 항목 금액이 숫자로 입력되고 통화가 별도 표시된다
- 합계가 자동 계산되고 원화 환산이 표시된다
- 도넛 차트로 예산 분포가 시각화된다
- 편집 시 실시간으로 합계/비율이 업데이트된다
- 기존 여행 데이터가 정상 로드된다 (하위 호환)
